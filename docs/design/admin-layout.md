# 管理页布局设计

> 本文是 [internal/admin/assets/index.html](../../internal/admin/assets/index.html) 的布局重构 spec。
>
> **状态（2026-07-19）**：布局与视觉重构已落地。预览截图见同目录 admin-*.png。

## 一、目标与约束

当前管理页观测台采用 `workbench` 网格在宽屏左右分栏（左主表 + 右聚合），配置页用顶部 `tabs` 切换四个子页。用户反馈：

1. 观测台**禁止左右布局**，改为纯纵向三段。
2. 配置页改为**左侧菜单**驱动。
3. 配置输入框（供应商、全局参数）**两个一行、等宽、撑满所在区域**。
4. 引导语 textarea **宽度和高度都撑满剩余区域**。
5. Topbar 承担观测台 / 配置页切换；去掉实时时钟和"重连"按钮。

非功能约束沿用 AGENTS.md：管理页是旁路，不得阻塞 `/v1/*`；保持 `recoverMiddleware` 包裹；daisyUI 5 + Tailwind 4 + Alpine 3 技术栈不变；i18n（zh/en）与主题切换保留。

## 二、Topbar

```
┌──────────────────────────────────────────────────────────────────┐
│ ●  codex-api-gateway  控制台    [观测台][配置]        🌐  ☾/☀   │
└──────────────────────────────────────────────────────────────────┘
  ↑ SSE 心跳                ↑ 页面切换 tabs           ↑ 语言/主题
```

保留：SSE 心跳点（`pulse` signature 动画）、品牌字、页面切换、语言切换、主题切换。

移除：中段实时时钟（`now` / `formatTime`）、"重连"按钮（`reconnect()`）。SSE 断线靠心跳点颜色变化即可表达，不需要额外按钮--`EventSource` 自带重连，手动重连是冗余操作。

实现上：把当前 `<nav>` 里的两个 tab（observe / config）合并进 `<header>`，靠右排在语言切换左侧。`<nav>` 整条删除。

## 三、观测台：纵向三段

```
┌──────────────────────────────────────────────────────────────────┐
│ Topbar                                                           │
├──────────────────────────────────────────────────────────────────┤
│ ① 总览：KPI 带（6 个 stat，2/3/6 列响应式）                      │
├──────────────────────────────────────────────────────────────────┤
│ ② 供应商 × 模型：聚合表（横向铺满，纵向按内容高度）              │
├──────────────────────────────────────────────────────────────────┤
│ ③ 请求历史：虚拟滚动表                                          │
│    flex: 1 1 auto; min-height: 420px;                           │
│    撑满视口剩余高度，窄屏/矮屏用 min-height 兜底                 │
│                                                                  │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

容器用 `flex flex-col` + `h-[calc(100vh-4rem)]`（4rem = topbar 高度），让 ③ 能拿到剩余高度。

- ① KPI 带：保留现有 `grid grid-cols-2 sm:grid-cols-3 xl:grid-cols-6`，不改。
- ② 聚合表：从右侧栏拆出来，改成全宽 `card`，内容不变（source/model/requests/cache_hit/avg_ms 五列）。`max-h-[520px]` 改为按内容自适应（去掉固定高度，因为纵向流里它不需要独立滚动）。
- ③ 请求历史：`flex-1 min-h-[420px] flex flex-col`。表头 + 过滤器固定，虚拟滚动区 `flex-1 min-h-0 overflow-auto`。`min-h-[420px]` 是兜底--当视口很矮（如手机横屏）时至少能显示约 10 行。

窄屏（< 768px）：三段自然堆叠，③ 的 `min-h-[420px]` 生效；宽屏（≥ 1280px）：① 一行 6 列、② 全宽、③ 拿到 `100vh - topbar - ① - ②` 的剩余高度。

## 四、配置页：左侧菜单 + 内容区

```
┌──────────────────────────────────────────────────────────────────┐
│ Topbar                                                           │
├────────────┬─────────────────────────────────────────────────────┤
│  供应商    │  内容区（每个子页独立滚动）                          │
│  模型      │                                                     │
│  全局参数  │  供应商 / 全局参数：输入框两列等宽                   │
│  引导语    │  模型：表格                                         │
│            │  引导语：textarea 撑满                              │
│            │                                                     │
│            │  ┌─────────────────────┐  ┌─────────────────────┐   │
│            │  │ 源名称              │  │ 上游地址            │   │
│            │  └─────────────────────┘  └─────────────────────┘   │
│            │  ┌─────────────────────┐  ┌─────────────────────┐   │
│            │  │ API 密钥（跨两列）  │  │                     │   │
│            │  └─────────────────────┘  └─────────────────────┘   │
│            │                                                     │
│            │  [sticky 底部：放弃修改 / 保存并热重载]              │
└────────────┴─────────────────────────────────────────────────────┘
```

左侧菜单：daisyUI `menu` 组件，垂直排列四个条目（sources / models / global / guidance）。宽屏固定 `w-48` 或 `w-56`，窄屏（< 768px）折叠成顶部横向 `tabs`（窄屏空间不够放左侧菜单）。

容器：`flex` 横向，菜单 `shrink-0`，内容区 `flex-1 min-w-0 overflow-y-auto`。整页高度 = `100vh - topbar`，内容区独立滚动，保存条 sticky 贴内容区底部。

### 4.1 输入框两列等宽规则（供应商、全局参数）

每个 `input` / `select` 包一层 `flex flex-col gap-1`，外层用 `grid grid-cols-2 gap-4`。规则：

- **默认两列等宽**：`grid-cols-2`，两列宽度由 grid 自动平分，输入框 `w-full`。
- **跨列字段**（如 API 密钥、日志文件、监听地址等长字段）用 `md:col-span-2`，单独占一行，宽度撑满。
- **窄屏单列**：`grid-cols-1 md:grid-cols-2`，窄屏自动堆叠。
- **大小一致**：全部用 `input-sm` / `select-sm`，不混用 `input-xs`。字体 `font-mono` 用于 URL / 密钥 / 模型名等机器字段。

供应商卡内的字段网格示例（语义，非最终代码）：

| 字段             | 列宽       | 说明                          |
|------------------|------------|-------------------------------|
| 源名称           | 1 列       |                               |
| 上游地址         | 1 列       |                               |
| API 密钥         | 跨 2 列    | 长字段，配显示/隐藏切换       |
| 默认模型         | 跨 2 列    | 带 datalist 上游模型补全      |
| 模型映射         | 跨 2 列    | fieldset，内部 join 行        |

全局参数两个 fieldset（全局参数、断路器）各自 `grid grid-cols-1 md:grid-cols-2 gap-4`，字段按语义两两配对：监听地址 / 缓存时效、日志级别 / 日志格式、日志文件（跨列）、首字节超时 / 冷却时间、降级阈值 / 恢复阈值、半开探测数 / 最大重试、恢复策略（跨列或配对）。

### 4.2 模型子页

保持现有表格（slug / context_window / supports_image / actions）。表格 `w-full`，新增模型走 modal 表单（不变）。

### 4.3 引导语子页

```
┌────────────┬─────────────────────────────────────────────────────┐
│  ...       │  引导语                              [重新载入]      │
│  引导语 ←  │  /path/to/base_instructions.md                      │
│            │  ┌───────────────────────────────────────────────┐  │
│            │  │                                               │  │
│            │  │  textarea                                     │  │
│            │  │  flex-1, min-h-0                              │  │
│            │  │  撑满剩余高度和宽度                            │  │
│            │  │                                               │  │
│            │  └───────────────────────────────────────────────┘  │
│            │  [保存引导语]                                      │
└────────────┴─────────────────────────────────────────────────────┘
```

引导语容器 `flex flex-col h-full`，textarea `flex-1 min-h-0 w-full`。这样无论窗口多大，textarea 都填满内容区剩余空间，不再用固定 `min-h-[420px]`。

## 五、保存条（sticky bottom）

当前保存条用 `sticky bottom-4`。新布局下：

- 供应商 / 模型 / 全局参数 三个子页：保存条 sticky 贴内容区底部，`position: sticky; bottom: 0`，带 `bg-base-100/95 backdrop-blur` 保证滚动时内容不穿帮。
- 引导语子页：不用全局保存条，保存按钮就在 textarea 下方（已有）。

## 六、响应式断点

| 断点          | Topbar 切换 | 观测台     | 配置页菜单   |
|---------------|-------------|------------|--------------|
| < 768px       | tabs 收紧   | 三段堆叠   | 折叠成顶部 tabs |
| 768–1280px    | tabs        | 三段，KPI 3 列 | 左侧菜单 w-48 |
| ≥ 1280px      | tabs        | 三段，KPI 6 列 | 左侧菜单 w-56 |

## 七、移除清单

- `<nav>` 整条（topbar 下方的 tab 导航条）--tabs 上移到 topbar。
- Topbar 中段实时时钟（`now` / `formatTime` / `setInterval`）。
- "重连"按钮及 `reconnect()` 方法。`EventSource` 自带重连，心跳点颜色已足够表达连接状态。
- `.workbench` CSS 规则（宽屏左右分栏）。
- 聚合表的 `max-h-[520px]`（纵向流里不需要独立滚动）。

`now` 状态和 timer 可以保留给其他用途，但若没有其他消费者就一并删掉。

## 八、不改的部分

- SSE 事件流（`/admin/api/events`）与 snapshot 数据结构。
- 虚拟滚动逻辑（`ROW_H` / `viewStart` / `viewCount` / `onListScroll`）。
- 后端 API（`/admin/api/*`）。
- i18n 字典（zh/en），只删除时钟和重连对应的 key（`clock` / `reconnect` / `live` / `offline` 中不再用的）。
- 主题切换、`pulse` signature 动画。
- modal / toast 机制。

## 九、验收标准

- [x] Topbar 包含观测台/配置切换，无实时时钟，无重连按钮。
- [x] 观测台三段纯纵向，宽屏无左右分栏，请求历史撑满剩余高度且 `min-height: 420px`。
- [x] 配置页左侧菜单（宽屏），窄屏折叠成顶部 tabs。
- [x] 供应商、全局参数子页输入框两列等宽，跨列字段用 `col-span-2`。
- [x] 引导语 textarea 撑满内容区剩余宽高。
- [x] 保存条 sticky 贴内容区底部（引导语子页除外）。
- [x] `go test ./internal/admin/...` 通过；管理页行为测试不回归。

## 十、视觉主题（落地补充）

- 冷调近白 / 深石墨底，靛蓝 primary + 青绿 accent（心跳色）
- KPI 卡片顶部 3px 语义色条（按指标动态 `primary/secondary/info/success/accent`）
- 玻璃顶栏 + primary 微底边线；菜单 active 左侧 3px 指示条

## 十一、组件体系（无 UI 库重构）

已移除 daisyUI / Tailwind CDN，改为自建 `ui-*` 组件系统（Phosphor Console）：

- token：`--ink / --paper / --surface / --signal / --focus / --good`
- 组件：`ui-topbar` `ui-tabs` `ui-btn` `ui-field` `ui-card` `ui-kpi` `ui-table` `ui-log` `ui-menu` `ui-modal` `ui-toast`
- 布局：观测台纵向三段；配置页左侧菜单；输入框两列等宽；引导语撑满剩余区域
- 唯一 signature：SSE 心跳 `ui-pulse`
